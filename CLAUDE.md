# CLAUDE.md — 小游戏合集项目规范

> 本文件是 Claude Code 的项目级指令文件，所有开发操作必须遵守以下规范。

---

## 一、项目概述

- **名称**：小游戏合集
- **技术栈**：纯前端，单文件 HTML + 内嵌 CSS/JS，零外部依赖
- **运行方式**：双击 HTML 文件直接在浏览器运行
- **语言**：简体中文界面

## 二、目录结构

```
小游戏/
├── CLAUDE.md           # 本文件（项目级开发规范）
├── index.html          # 游戏大厅（主页，星空粒子动画 + 卡片布局）
├── README.md
├── CHANGELOG.md        # 根目录变更日志（跨游戏变更）
├── _template/          # 新游戏开发模板
│   ├── index.html      # 含 A~G 开发前置清单
│   └── CHANGELOG.md
├── 贪吃蛇/
├── 俄罗斯方块/
├── 踩雷/
├── 换装游戏/
└── 五子棋/
```

每个游戏文件夹包含：
- `index.html` — 游戏主文件（唯一文件）
- `CHANGELOG.md` — 该游戏的独立开发日志

---

## 三、开发日志规范（强制）

**每次修改任何游戏代码，必须同步更新该游戏的 CHANGELOG.md。**

### 格式要求
```markdown
## [版本号] - YYYY-MM-DD

### Added（新增）
- 新功能描述

### Changed（变更）
- 修改内容描述

### Fixed（修复）
- 修复内容描述
```

### 版本号规则
| 变更类型 | 版本号变化 | 示例 |
|---------|-----------|------|
| 新功能 | minor +1 | 1.0.0 → 1.1.0 |
| Bug 修复 | patch +1 | 1.1.0 → 1.1.1 |
| 大重构 | major +1 | 1.1.1 → 2.0.0 |

### 日志归属
- 单个游戏的修改 → 该游戏文件夹的 `CHANGELOG.md`
- 跨游戏变更（大厅升级、全局规范调整） → 根目录 `CHANGELOG.md`

## 四、响应式设计规范（强制）

**所有页面必须适配以下设备：**

| 设备 | 断点 | 要求 |
|------|------|------|
| 手机 | ≤480px | 紧凑布局，单列卡片，缩小字体 |
| 平板 | ≤768px | 中等布局，双列卡片 |
| 桌面 | ≥769px | 完整布局 |
| 触屏 | `@media (pointer: coarse)` | 显示虚拟控制按钮 |

### 必须包含
1. `<meta name="viewport" content="width=device-width, initial-scale=1.0">`
2. CSS 媒体查询：`@media (max-width: 768px)` + `@media (max-width: 480px)`
3. 触屏适配：`@media (pointer: coarse)` 显示虚拟控制

### 按游戏类型
| 类型 | 触屏适配方案 |
|------|------------|
| Canvas 方向游戏（贪吃蛇、俄罗斯方块） | 虚拟方向键 + 功能键，Canvas 添加 `touch-action: none` |
| 点击类游戏（踩雷） | 「插旗模式」切换按钮替代右键，cell 尺寸动态计算 |
| 棋盘类游戏（五子棋） | cell/棋盘尺寸根据 viewport 动态计算，touch 事件落子 |
| 点击交互类（换装） | onclick 自然兼容触屏，布局堆叠适配 |

## 五、技术规范

### 5.1 文件格式
- 每个游戏 **只有一个 index.html 文件**
- 所有 CSS 写在 `<style>` 标签内
- 所有 JS 写在 `<script>` 标签内
- **禁止引用外部 CDN、字体、图片等任何外部资源**

### 5.2 CSS 设计变量
每个游戏使用 `:root` 定义设计变量，必须包含主题色 `--accent`。

已分配主题色：
| 游戏 | 主题色 | 色名 |
|------|-------|------|
| 贪吃蛇 | `#00ff88` | 霓虹绿 |
| 俄罗斯方块 | `#e94560` | 热情红 |
| 踩雷 | `#ffd54f` | 警戒黄 |
| 换装小屋 | `#ff85c2` | 玫瑰粉 |
| 五子棋 | `#4fc3f7` | 天蓝 |

新游戏必须选择不与已有游戏冲突的主题色。

### 5.3 localStorage 约定
```
键名格式：小游戏_[游戏名]_[字段]
全局静音：小游戏_全局_静音      （布尔值，跨游戏共享）
总局数：  小游戏_[游戏名]_总局数  （整数，供大厅卡片显示）
最高分：  小游戏_[游戏名]_最高分  （视游戏而定）
```

### 5.4 音效系统
- 使用 Web Audio API 合成音效，**禁止加载外部音频文件**
- 参考 `_template/index.html` 中的 SFX 对象实现
- 全局静音状态读取 `小游戏_全局_静音`
- 每个游戏页面提供静音切换按钮

### 5.5 导航
- 每个游戏页面左上角必须有 `← 游戏大厅` 返回链接
- 链接地址：`../index.html`

## 六、新游戏开发流程

### 第1步：确认需求清单（A~G）

开发前必须向用户确认以下所有信息（详见 `_template/index.html`）：

- **A. 基本信息**：名称、简介、emoji、主题色
- **B. 核心玩法**：规则、胜利条件、失败条件、难度等级
- **C. 计分与进度**：计分规则、等级系统、存储需求、计时器
- **D. 视觉与技术**：Canvas/DOM 选择、游戏区域尺寸、特效需求
- **E. 操作方式**：键盘/鼠标/触屏方案
- **F. 可选功能**：音效、动画、暂停、历史最佳、连击机制
- **G. 系统级规则**：CHANGELOG、响应式适配、触屏控制

### 第2步：创建游戏文件
1. 复制 `_template/` 文件夹，重命名为游戏中文名
2. 基于模板开发游戏逻辑
3. 创建 `CHANGELOG.md` 记录 v1.0.0

### 第3步：注册到游戏大厅
在 `index.html` 的 `GAMES` 数组末尾追加：
```javascript
{
  id:         '游戏名',
  title:      '游戏名',
  emoji:      '🎮',
  desc:       '一句话描述',
  tag:        '分类标签',
  accent:     '#主题色',
  glowRgb:    'R,G,B',
  path:       '游戏名/index.html',
  bestKey:    '小游戏_游戏名_最高分',
  bestLabel:  '最高分',
  bestUnit:   '分',
  countKey:   '小游戏_游戏名_总局数',
}
```

### 第4步：更新 CHANGELOG
- 游戏文件夹 `CHANGELOG.md`：记录 v1.0.0
- 根目录 `CHANGELOG.md`：记录新游戏上线
